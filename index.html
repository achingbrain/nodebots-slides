What's a NodeBot?

node.js + serial port + firmata + johnny-five

  • Nodebots are robots powered by JavaScript
  • Controlled via a virtual serial port
  • Use the firmata protocol
  • Johnny-Five node.js library
  • Built with esoteric hardware, such as...

Arduino

  • Open source electronics prototyping platform
  • Digital and analogue input and output pins
  • Cheap and accessible

Arduino IDE

  • The lovechild of Processing and Wiring
  • Program in a subset of C
  • Only used to flash Arduino with Firmata

Firmata

  • Protocol and firmware for exposing Arduino capabilities to host computer
  • Turn outputs on and off
  • Read from inputs
  • Looks a bit like MIDI
  • People build crazy robots
  • Arduino on the back
  • Sonar sensor on the front
  • Slightly more sinister

B.A.R.B.A.R.A.

Build A Robot, Brew A Real Ale

A micro-services approach to brewing beer

The idea

  • Constant temperature key to repeatable brew
  • Measure temperature
  • Constant temperature adjustments

Hardware

Rasperry-Pi

BeagleBone Black

2x Arduinos

Temperature Sensor

Aquarium Heater

Micro services

Single-responsibility processes

  • Temperature sensor
  • Heater
  • Database
  • Twitter
  • Service registry

Service location

  • Seaport - https://github.com/substack/seaport
  • mDNS - Bonjour/Zeroconf/Avahi
  • Temperature sensor will know it's monitoring ipa#1 and needs to report the temperature to a database
  • Heater will know it needs the temperature of ipa#1 so will find it on the network
  • Seaport contains which service on which host/port
  • Locate services through Seaport
  • Locate Seaport through mDNS
  • When Seaport goes down and comes back everything reconnects

Fault tolerance

pm2 - https://github.com/Unitech/pm2

  • Each services has it's own process
  • Each process initialises whatever it needs
  • pm2 keeps processes running
  • Could run clustered if so inclined
